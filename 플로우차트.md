# EWA 영어 학습 앱 플로우 차트

## 1. 전체 사용자 여정 (Overall User Journey)

```mermaid
graph TD
    A[앱 실행] --> B{로그인 상태?}
    B -->|미로그인| C[로그인/회원가입]
    C --> D[레벨 선택]
    D --> E[메인 대시보드]
    B -->|로그인됨| E
    
    E --> F{어떤 활동?}
    F -->|책 읽기| G[책 목록]
    F -->|단어 복습| H[어휘장]
    F -->|통계 확인| I[분석 페이지]
    
    G --> J[책 선택]
    J --> K[읽기 화면]
    H --> L[복습 모드]
    I --> M[통계 & 차트]
    
    K --> N{종료?}
    N -->|계속| K
    N -->|종료| E
    
    L --> O{복습 완료?}
    O -->|계속| L
    O -->|완료| E
```

## 2. 책 읽기 상세 플로우 (Book Reading Flow)

```mermaid
graph TD
    A[읽기 시작] --> B[현재 페이지 로드]
    B --> C[원문 + 번역 표시]
    
    C --> D{사용자 액션?}
    
    D -->|단어 드래그| E[단어 선택 감지]
    E --> F[팝업: 정의 표시]
    F --> G{추가?}
    G -->|YES| H[어휘장에 저장]
    H --> I[팝업 닫기]
    G -->|NO| I
    I --> C
    
    D -->|TTS 버튼| J[문장 오디오 재생]
    J --> K{재생 중?}
    K -->|YES| L[진행 상황 표시]
    L --> K
    K -->|NO| C
    
    D -->|번역 토글| M[번역 보이기/숨기기]
    M --> C
    
    D -->|북마크| N[현재 문장 북마크]
    N --> C
    
    D -->|다음 페이지| O[다음 페이지 로드]
    O --> B
    
    D -->|이전 페이지| P[이전 페이지 로드]
    P --> B
    
    D -->|종료| Q[진도 저장]
    Q --> R[읽기 화면 닫기]
```

## 3. 단어 학습 상세 플로우 (Word Learning Flow)

```mermaid
graph TD
    A[사용자가 단어 드래그] --> B[드래그 범위 감지]
    B --> C[단어 추출 & 정규화]
    C --> D[사전에서 검색]
    
    D --> E{검색 결과?}
    E -->|찾음| F[정의 팝업 표시]
    E -->|못 찾음| G[사용자에게 알림]
    G --> H[앱 종료]
    
    F --> I{사용자 선택?}
    I -->|정의 보기| J[영문 정의 + 한글 번역 표시]
    J --> I
    
    I -->|발음 듣기| K[IPA 및 오디오 재생]
    K --> I
    
    I -->|예문 보기| L[책 내 출현 위치 표시]
    L --> I
    
    I -->|추가| M[UserVocabulary 테이블에 저장]
    M --> N[추가 확인 메시지]
    N --> O[팝업 닫기]
    
    I -->|닫기| O
    O --> P[읽기 화면으로 돌아가기]
```

## 4. 복습 시스템 플로우 (Vocabulary Review Flow)

```mermaid
graph TD
    A[어휘장 탭] --> B[복습 필요 단어 필터링]
    B --> C[복습 모드 시작]
    
    C --> D{복습 타입 선택?}
    
    D -->|플래시카드| E[앞면: 영문]
    E --> F{카드 뒤집기?}
    F -->|YES| G[뒷면: 한글 + 정의]
    G --> H{단어 알기?}
    H -->|알음| I[정답 표시]
    H -->|모름| J[오답 표시]
    
    D -->|선택지 문제| K[선택지 4개 제시]
    K --> L[사용자 선택]
    L --> M{정답?}
    M -->|YES| I
    M -->|NO| J
    
    D -->|듣기 테스트| N[음성 재생]
    N --> O[선택지 중 선택]
    O --> M
    
    I --> P[복습 카운트 증가]
    J --> Q[오답 표시, 다음 복습 예약]
    
    P --> R{더 복습?}
    Q --> R
    
    R -->|YES| S{복습 타입?}
    S --> D
    
    R -->|NO| T[통계 업데이트]
    T --> U[진도율 저장]
    U --> V[복습 완료 화면]
```

## 5. 회원가입 & 초기화 플로우 (Onboarding Flow)

```mermaid
graph TD
    A[앱 시작] --> B[로그인 여부 확인]
    B -->|미로그인| C[로그인 화면]
    
    C --> D{회원가입?}
    D -->|로그인| E[이메일/비번 입력]
    D -->|회원가입| F[이메일/비번 입력]
    
    E --> G{로그인 성공?}
    F --> H{가입 성공?}
    
    G -->|NO| I[에러 메시지]
    I --> E
    
    H -->|NO| J[에러 메시지]
    J --> F
    
    G -->|YES| K[메인 대시보드]
    H -->|YES| L[레벨 선택 화면]
    
    L --> M[초급/중급/고급 선택]
    M --> N[맞춤 책 추천]
    N --> O[첫 번째 책 추천]
    O --> K
```

## 6. 데이터 플로우 (Data Flow)

```mermaid
graph LR
    A[클라이언트/React] -->|API 요청| B[Node.js 백엔드]
    B -->|쿼리| C[(PostgreSQL)]
    B -->|캐시| D[(Redis)]
    B -->|저장| E[S3/R2<br/>책 파일, TTS]
    
    C -->|응답| B
    D -->|응답| B
    E -->|제공| B
    
    B -->|응답| A
    
    A -->|IndexedDB| F[로컬 저장소<br/>오프라인용]
    F -->|읽기| A
```

## 7. TTS 시스템 플로우 (Text-to-Speech Flow)

```mermaid
graph TD
    A[사용자가 TTS 버튼 클릭] --> B[현재 문장 추출]
    B --> C{캐시에 있는가?}
    
    C -->|YES| D[캐시된 오디오 재생]
    C -->|NO| E[TTS 서비스에 요청]
    
    E --> F{생성 성공?}
    F -->|YES| G[오디오 파일 생성]
    G --> H[S3/R2에 저장]
    H --> I[Redis 캐시에 저장]
    I --> J[오디오 재생]
    
    F -->|NO| K[에러 처리]
    K --> L[사용자에게 알림]
    
    D --> M[음성 재생 중]
    J --> M
    M --> N{재생 완료?}
    N -->|YES| O[완료]
    N -->|일시정지| P[사용자 제어]
    P --> M
```

## 8. 오프라인 모드 플로우 (Offline Mode Flow)

```mermaid
graph TD
    A{인터넷 연결?}
    
    A -->|연결됨| B[온라인 모드]
    B --> C[서버와 동기화]
    C --> D[최신 데이터 로드]
    D --> E[읽기/복습]
    
    A -->|연결 안 됨| F[오프라인 모드]
    F --> G{필요한 책이<br/>다운로드됨?}
    
    G -->|YES| H[IndexedDB에서 로드]
    H --> I[로컬 데이터로 읽기]
    I --> J[변경사항 저장<br/>로컬에만]
    J --> K{인터넷 복구?}
    
    G -->|NO| L[다운로드 필요 알림]
    
    K -->|YES| M[동기화 시작]
    M --> E
    K -->|NO| N[계속 오프라인]
```

## 9. 복습 스케줄 알고리즘 (Spaced Repetition Schedule)

```mermaid
graph TD
    A[단어 추가] --> B[상태: NEW]
    B --> C[첫 복습: 즉시]
    
    C --> D{복습 결과?}
    
    D -->|정답| E[상태: LEARNING]
    E --> F[다음 복습: +1일]
    
    D -->|오답| G[상태: NEW]
    G --> F
    
    F --> H[사용자 복습함]
    H --> I{다시 정답?}
    
    I -->|YES| J[다음 복습: +3일]
    J --> K[상태 유지]
    
    I -->|NO| L[다음 복습: +1일]
    L --> K
    
    K --> M[또 복습함]
    M --> N{또 정답?}
    N -->|YES| O[다음 복습: +7일]
    N -->|NO| P[다음 복습: +1일]
    
    O --> Q[상태: MASTERED]
    P --> K
```

## 10. 에러 처리 플로우 (Error Handling Flow)

```mermaid
graph TD
    A[오류 발생] --> B{오류 타입?}
    
    B -->|네트워크 오류| C[오프라인 알림]
    C --> D[로컬 데이터 사용]
    
    B -->|API 오류| E[재시도 로직]
    E --> F{재시도 성공?}
    F -->|YES| G[계속 진행]
    F -->|NO| H[사용자 알림]
    H --> I[대안 제시]
    
    B -->|DB 오류| J[관리자 알림]
    J --> K[사용자에게<br/>일시적 문제 알림]
    
    B -->|TTS 오류| L[웹 TTS로 폴백]
    L --> M[텍스트로 표시]
```

## 11. 모바일 & 데스크톱 동기화 (Cross-Device Sync)

```mermaid
graph TD
    A[모바일에서 읽기 시작] --> B[진도 저장]
    B --> C[서버에 동기화]
    C --> D[(클라우드)]
    
    D --> E[데스크톱 앱 열기]
    E --> F[마지막 위치에서<br/>재개 제안]
    F --> G{계속?}
    
    G -->|YES| H[서버에서<br/>진도 동기화]
    G -->|NO| I[로컬 진도 사용]
    
    H --> J[최신 어휘장 로드]
    I --> K[데스크톱 로컬 데이터 사용]
    
    J --> L[읽기 계속]
    K --> L
```
